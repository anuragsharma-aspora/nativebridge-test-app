## üìã Quick Reference

### Adding More Ignored Files

To ignore additional files, add to `.gitignore`:

```bash
# Example: Ignore all .secret files
*.secret

# Example: Ignore specific directory
/sensitive-data/

# Example: Ignore file in specific location only
android/app/production-config.json
```

### Checking Ignored Files

```bash
# List all ignored files
git status --ignored

# Check if a file is ignored
git check-ignore android/app/nativebridge-release.keystore

# Show why a file is ignored
git check-ignore -v android/app/nativebridge-release.keystore
```

### If You Accidentally Committed Secrets

```bash
# Remove from git but keep file locally
git rm --cached <file>
git commit -m "Remove sensitive file from git"

# Remove from entire git history (DANGEROUS - rewrite history)
git filter-repo --path <file> --invert-paths
```

## üéØ Best Practices

1. **Before First Commit**
   - Review `.gitignore`
   - Run `git status` to check staged files
   - Search for secrets: `git grep -i password`
   - Use `git add -p` for careful staging

2. **Regular Reviews**
   - Update `.gitignore` as project grows
   - Check for new sensitive files
   - Audit what's tracked vs ignored
   - Team review of committed files

3. **Team Onboarding**
   - Share this summary with new developers
   - Explain why files are ignored
   - Review [SECURITY.md](SECURITY.md) together
   - Set up pre-commit hooks if needed
